# 智能净水器：水泵电流监测与自适应保护架构说明

## 一、 硬件测量模型（底层物理映射）

系统的底层核心逻辑是将大电流转化为 ESP32 可安全读取的微电压信号。

* **硬件参数提取**：
    * 低边采样电阻：采用 3 颗 $150m\Omega$ 贴片电阻并联，总阻值 $R_{shunt} = 50m\Omega$ ($0.05\Omega$)。
    * 运放放大倍数：同相放大器（反馈 $82k\Omega$，接地 $10k\Omega$），增益 $A_v = 1 + (82/10) = 9.2$ 倍。
* **绝对转换公式**：
    到达 ESP32 IO39 引脚的电压为：$V_{ADC} = I_{pump} \times 0.05 \times 9.2$
    即：**$V_{ADC} = 0.46 \times I_{pump}$**
* **边界评估**：即使在极端死锁（$3.5A$）情况下，最高输入电压仅为 $1.61V$。该设计完美避开了 ESP32 ADC 在 $2.5V$ 以上的非线性盲区。

## 二、 软件采集与精度优化（ADC 调优）

为了将硬件精度发挥到极致，在固件驱动层（`bsp_sensor.c`）实施了针对性的优化：

1.  **动态量程匹配（核心亮点）**：
    摒弃了默认的 `11dB` 衰减配置。针对水泵通道最高 $1.61V$ 的特性，独立配置为 **`ADC_ATTEN_DB_6`（6dB 衰减）**。这使得该通道的量程缩小至约 $0 \sim 1.75V$，完美包裹水泵电压极值，在不改动硬件的前提下将采样分辨率直接翻倍。
2.  **均值滤波防抖**：
    引入了连续 10 次的滑动采样求平均（Moving Average），有效过滤掉 LDO 纹波与底噪，为毫安级的“空转判定（0.5A）”提供稳定数据源。

## 三、 自适应识别与状态机保护（业务逻辑）

逻辑层（`app_fsm.c`）结合水机运行状态，实现了“免配置、自识别、抗浪涌”的保护机制：

1.  **无感硬件自适应（Auto-Recognition）**：
    利用开机首次制水/冲洗的第 5 秒（完美避开启动浪涌）进行平稳期采样：
    * 测得电流 $< 1.8A$：系统自适应为 **100G 规格**，本周期过流阈值锁定为 $2.0A$。
    * 测得电流 $\ge 1.8A$：系统自适应为 **400G 规格**，本周期过流阈值锁定为 $3.0A$。
2.  **滑动窗口异常保护**：
    * **过流/堵转**：实时电流 $>$ 动态阈值，且连续维持 **2 秒**，触发致命异常停机。
    * **空转/缺水**：实时电流 $< 0.5A$，且连续维持 **3 秒**，触发异常停机。

## 四、 下一步迭代计划（给 AI / 工程师的 Review 重点）

当前的计算模型基于理想的线性公式（`raw * 2.2 / 4095`），为了应对 ESP32 芯片体质带来的 $V_{ref}$（内部基准电压）批次误差，下一阶段的重构重点是引入出厂校准库。

**专项优化任务清单（TODO）：**

1.  **引入 `esp_adc_cal` / `adc_cali` 校准模块**：
    废弃代码中的硬编码电压换算。在初始化阶段，要求读取芯片内部 eFuse 烧录的特征曲线（Curve Fitting）。
2.  **直接提取高精度 mV 值**：
    使用官方校准接口（如 `adc_cali_raw_to_voltage`）获取真实的毫伏值，再代入原理图公式 $I_{pump} = V_{mV} / 460$ 计算真实电流。
3.  **零点漂移抹除（Zero-drift Cancellation）**：
    在每次水泵启动前（关机状态），采样一次底噪电压作为“环境零点基准”。后续计算中先扣除该基准，以抵消运放芯片的输入失调电压，实现医疗级的采集精度。
4.  **NVS 交叉校验机制**：
    将自适应识别的“水泵规格（100G/400G）”持久化写入 NVS。下次开机自适应时，优先比对历史记录，若出现严重背离（如原本装的 400G 突然测出 100G），则抛出硬件老化或异常告警。
